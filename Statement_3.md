## 1. Problem Statement
Hospitals need a simple system to record and manage basic patient information.  
Many small hospitals still depend on manual registers, which can cause  
mismanagement and loss of records. This project aims to provide a simple  
digital solution to store and manage patient details efficiently.

## 2. Scope of the Project
- Store patient details such as name, age, and disease  
- View all saved patient records  
- Update or delete a patient using their ID  
- Provide a basic CRUD system for
educational and small-scale use  
- Console-based, simple, and easy to understand for beginners  

## 3. Target Users
- Students learning Python  
- Beginners practicing CRUD operations  
- Small clinics/hospitals needing basic patient record management  
- Teachers evaluating basic Python projects  

## 4. High-Level Features
- Auto-increment patient ID  
- Dynamic list-based data storage  
- Clear menu-driven interface
